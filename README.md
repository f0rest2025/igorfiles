# Yandex Object Storage Manager

Локальное web-приложение для оператора, который принимает файлы от клиентов в Yandex Object Storage и выдаёт безопасные временные ссылки. Приложение запускается на Linux и Windows, слушает `127.0.0.1` и не требует Docker, внешней БД или регистрации пользователей.

## Архитектура

Выбран FastAPI + статический web-GUI. Это практичный вариант для MVP: один Python-процесс, нормальный интерфейс в браузере, простая поддержка и одинаковый запуск на Windows/Linux.

Операторские ключи используются только локальным backend. Клиентская ссылка создаётся для одного object key и ограничена сроком жизни. В интерфейсе выдаются:

- клиентская HTML-ссылка `data:` для прямой загрузки через browser `PUT` по pre-signed URL;
- локальная страница `/upload/<token>`, которая загружает файл через уже созданный pre-signed URL и не показывает список объектов или ключи;
- raw pre-signed `PUT` URL для технических сценариев.

Важно: локальная страница `/upload/<token>` доступна клиенту только там, где доступен запущенный backend. По умолчанию backend слушает только `127.0.0.1`, как требует ТЗ. Для внешних клиентов используйте `data:` HTML-ссылку или raw pre-signed URL; для прямой browser-загрузки bucket должен разрешать CORS для `PUT`.

## Установка

### Windows 10/11

1. Установите Python 3.11+.
2. Откройте PowerShell в папке проекта.
3. Создайте окружение и установите зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

4. Запустите приложение:

```powershell
.\run_windows.bat
```

5. Откройте `http://127.0.0.1:8765`.

### Linux

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
sh run_linux.sh
```

Откройте `http://127.0.0.1:8765`.

## Локальный конфиг

Конфиг хранится отдельно от исходников:

- Windows: `%APPDATA%\YandexStorageFileManager\config.json`
- Linux: `~/.config/yandex-storage-file-manager/config.json`

Путь можно переопределить переменной окружения `YOS_MANAGER_CONFIG`.

Пример есть в `config.example.json`:

```json
{
  "access_key_id": "YANDEX_ACCESS_KEY_ID",
  "secret_key": "YANDEX_SECRET_KEY",
  "bucket": "my-bucket",
  "prefix": "incoming",
  "endpoint": "https://storage.yandexcloud.net",
  "region": "ru-central1"
}
```

Secret Key не выводится обратно в интерфейс и не логируется приложением. Если поле Secret Key в GUI оставить пустым, уже сохранённый ключ не меняется.

## Основные сценарии

### Проверка подключения

Откройте вкладку «Подключение», заполните Access Key ID, Secret Key, Bucket, Prefix, Endpoint и Region. Нажмите «Проверить подключение». При успехе приложение выполняет `head_bucket`.

### Генерация upload-ссылки

Откройте «Ссылка на загрузку», укажите имя файла или object key, prefix, срок жизни и при необходимости Content-Type. Нажмите «Сгенерировать».

Результат содержит итоговый object key, клиентскую HTML-ссылку, локальную `/upload/<token>` страницу и raw pre-signed `PUT` URL. Все варианты привязаны к одному object key и ограничены временем.

### Клиентская загрузка

Передайте клиенту «Клиентскую HTML-ссылку для браузера». Она открывает минимальную форму выбора файла и выполняет `PUT` в Object Storage. Если браузер блокирует запрос из-за CORS, настройте CORS bucket для `PUT` или используйте локальную страницу `/upload/<token>` в доступной сети.

Клиентская страница не показывает список объектов, bucket-настройки, download-ссылки или ключи оператора.

### Список файлов

Откройте вкладку «Файлы», задайте prefix и нажмите «Обновить». Доступны поиск, сортировка, копирование object key и переход к генерации download-ссылки.

### Download-ссылка

Откройте «Ссылка на скачивание», укажите object key и срок жизни. Сгенерированная pre-signed `GET` ссылка показывается только в операторском интерфейсе.

### Прямая загрузка оператором

Откройте «Прямая загрузка», выберите файл, prefix и итоговое имя объекта. После загрузки можно скопировать object key и открыть список файлов.

## Безопасность MVP

- backend по умолчанию слушает только `127.0.0.1`;
- access key и secret key не передаются клиентской upload-странице;
- клиентский token привязан к одному object key;
- клиентский token одноразовый на стороне локальной страницы;
- raw pre-signed URL ограничен временем и операцией `PUT`;
- нет удаления, переименования, регистрации, БД и телеметрии.

Ограничение S3 pre-signed `PUT`: сама raw-ссылка не является одноразовой и до истечения срока может перезаписать тот же object key. Для практической защиты включайте GUID в имени и задавайте короткий TTL.
